Seleção de Características ^TXDfWsTM

Um dos principais aspectos na construção de um bom modelo é a utilização de características (features) discriminantes.
- É comum encontrar situações em que centenas de características são utilizadas para alimentar um modelo. A adição de uma nova não significa uma melhora, e pode até piorar o desempenho do modelo.

A features selection é a tarefa de identificar e selecionar um subconjunto de características relevantes para um determinado problema, a partir do conjunto inicial.
    - Ideal: menor subconjunto necessário para resolver um dado problema.
    - Clássico: selecionar um subconjunto de M características a partir de N características, a qual M < N, de maneira a minimizar uma dada função objetivo.
    - Melhor desempenho: buscar um subconjunto que melhore o desempenho de um dado modelo.

Não é um problema trivial.
- Em problemas reais, features discriminantes não são conhecidas a priori.
- Características raramente são totalmente independentes.
- Duas características que individualmente não contribuem de forma relevante podem, quando analisadas em conjunto, revelar interações que aumentam o poder discriminativo do modelo.
    - Isso está ligado a presença de interações não lineares entre atributos.
    - Muitas técnicas avaliam atributos de forma individual, o que pode levar à exclusão de pares ou grupos de características sinergéticas (só úteis em conjunto). ^uJcBdxAm

Visão Geral ^QRx0RQfc

Um método de seleção de características deve utilizar um método de busca para encontrar um subconjunto M a partir de N características (espaço de busca: 2 ).
- Para cada solução encontrada, se faz uma avaliação.
- Existe um critério de parada e uma validação. ^doL8iRLf

Funções de Avaliação ^XeXHH507

Para julgar se um subconjunto é ótimo, devemos usar uma função de avaliação, e elas podem ser dividias em:
- Filter: independentes do algoritmo de aprendizagem
- Wrapper: dependente do algoritmo de aprendizagem
- Embedded: usa algoritmos capazes de avaliar o poder de discriminação de características durante o treinamento do modelo ^guNVus6U

Filter ^npAcyldI

Independentes de qualquer algoritmo de aprendizagem.
- Geralmente usados como forma de pré-processamento.
- Importância das características são medidas através de diversos testes estatísticos, por exemplo:
    - correlação, mutual information (medida de dependência mútua entre duas variáveis), variância, ANOVA, chi-quadrado, etc.
- Vantagens: rápido, independente do modelo, fácil de calcular.
- Desvantagens: ignora a relação entre as características. ^OLkndmbq

sklearn: SelectKBest ^GhslKa4g

Wrapper ^ASYOlHUD

Utiliza o modelo para avaliar o conjunto.
- O processo de seleção de características deve ser visto como um processo de aprendizagem, sendo assim, é importante utilizar uma base de validação para evitar over-fitting.
- Vantagens: pode capturar a interação entre características (dependendo do método de busca).
- Desvantagens: custo computacional mais alto. ^idGWUFD0

Efeito de características irrelevantes ^MNOTI121

Quanto características afetam os modelos de predição?
    - Depende do tipo do modelo, natureza das características e também da relação entre tamanho da base de dados e número de características (p : n ratio).
    - Exemplo: avaliação experimental com três tipos de classificadores (lineares, não lineares e árvores) e 20 características (Kuhn & Johnson, 2020). Duas bases de treinamento (n=500, 1000) e até 200 características irrelevantes adicionadas. ^NyFFucfv

Comentário sobre PCA ^PRmynjM0

A Análise de Componentes Principais (PCA) é uma técnica de transformação linear, que projeta dados de alta dimensionalidade em um espaço de menor dimensão, capturando o máximo possível da variância presente nos dados.
- PCA encontra novas variáveis (chamados de componentes principais) que são:
    - Combinações lineares das variáveis originais
    - Ortogonais entre si (sem correlação)
    - Ordenadas pela quantidade de variância explicada

- PCA não é um método de seleção, pois todas características devem ser extraídas. ^FRTvrbVO
