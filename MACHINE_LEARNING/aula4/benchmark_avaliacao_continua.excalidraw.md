Benchmark para ML ^lu7U4r03

1) Datasets padronizados ^93I1hX5Z

- Datasets públicos
    - UCI, Kaggle, etc. para ter resultados comparáveis.
- Quando estiver usando dados proprietários, garanta que foram pré-processados (data cleaning) e rotulados corretamente.
    - Sempre que possível disponibilize os dados, ou parte deles.
- Sempre use um mecanismo de validação para evitar enviesamento. ^QqlH0Q5r

2) Métricas ^tvL6d0Ay

- Classificação: acurácia, Precision, Recall, F1-Score, AUC-ROC.
    - evite acurácia para problemas desbalanceados.
- Regressão: MAE, MSE, RMSE, R .
- Clustering: Silhouette Score, DB Index, etc.
    - combinação de índices (NBClust) ^7x8BAkdA

2 ^5WPFFHey

3) Hiper-parâmetros ^8OGvj0wn

- Para comparar modelos de forma justa, otimize os hiper-parâmetros de forma sistemática.
    - Grid-search: busca utilizando um conjunto pré-definido de valores.
    - Busca aleatória: seleciona aleatoriamente valores dentro de um range.
    - Algoritmos de meta-heurística.
    - Otimização Bayesiana.
    - Frameworks (optuna, hyperopt) ^6Vn93GS0

Testes estatísticos ^sQEYNPEN

- Comparação de acurácia não é suficiente.
    - Testes de significância estatística indicam se as diferenças são relevantes.
        - Teste t
        - Wilcoxon signed-rank test
    - Mostrar que possíveis melhorias são reais ou foram encontradas por acaso. ^adjujuEt

Eficiência Computacional ^WpqcuNgl

- Tempo de treinamento
    - Quanto tempo demora pra treinar o modelo e encontrar os hiper-parâmetros?
    - Custo associado.

- Tempo de inferência
    - Quando tempo leva para fazer uma predição?
    - Crítico em problemas de tempo real.
    - O melhor modelo pode não satisfazer esse critério.

- Uso de memória
    - Sistemas embarcados. ^Fm2UIWfC

Compare com um baseline ^uDJKTeV9

- Sempre compare o novo modelo com um baseline inicial.
    - Se o novo modelo não apresenta uma melhora significativa, precisa ser melhorado.
- Compare com o Estado da Arte da literatura.
    - Talvez alguém já resolveu seu problema. ^UfYeg5wS

Reprodutibilidade ^qnwpBaN5

- Para uma comparação justa, garanta que seus experimentos possam ser reproduzidos.
    - fixe as "random seeds" (ex: 42 no sklearn)
- Documente todos os passos.
    - ML_Flow Tracking
- Disponibilize, na medida do possível, dados e modelos. ^a8A92TTj

Avaliação Contínua ^ADR7dn7U

É importante pois a grande maioria dos algoritmos de ML assume que os dados de treinamento e teste são independentes (uma observação não tem influência na outra) e identicamente distribuídos (mesma distribuição de probabilidade). ^gp4n6GyK

- Em que P(X,y) é a distribuição conjunta das características X e do alvo y.
- Quando essa hipótese não é mais verdadeira,
    - Concept drift
    - Desempenho do modelo piora ^Ia2hMrlZ

Concept Drift ^HNLuZ54S

Mudança de conceito: acontece quando o relacionamento entre as características (X) e a variável alvo (y) muda ao longo do tempo.
- Significa que o desempenho do modelo não será o mesmo para novos dados.

Existem alguns tipos: ^iFvi4tBa

Virtual ^npYyXUE7

A distribuição das características (X) muda, mas a fronteira de decisão continua igual.
    - aparecem dados não observados durante o treinamento ^VEeg5SZl

Real ^Rsw94B0r

A fronteira de decisão muda ao longo do tempo.
    - A mesma entrada, agora pertence a outra classe. ^CXg2IVoM

Abrupto ^VOLqs7US

A fronteira de decisão muda abruptamente, tornando o modelo anterior obsoleto.
- Fácil detecção, pois a queda de desempenho é abrupta.
- Ex: sistema de reconhecimento de face no COVID, com uso de máscaras. ^O3qezcr2

Gradual (Slow drift) ^T6aBONNQ

Um conceito C1 é gradualmente substituído por C2.
- Eles co-existem por um período.
- Ex: classificação de email, as pessoas gradualmente começam a usar novas gírias e emojis. ^uYqnWlSy

Incremental ^d3zmRX4l

É uma mudança contínua (evolução lenta da fronteira de decisão.
- Difícil de detectar, pois o modelo tem bom desempenho no curto prazo, mas os erros se acumulam no longo prazo.
- Ex: mercado financeiro, onde uma política de uma empresa pode afetar os preços das ações. ^rlbBLGSk

Recorrente ^Ktk5L1Xo

O conceito que desaparece, pode reaparecer no futuro.
- Ex: conceitos pré-COVID e pós-COVID. ^95250brc

Como lidar com o Drift? ^s9JXP5vy

Abordagens ativas (detectar e reagir)
    - utilizar um método de detecção de drift e então atualizar o modelo.
        - DDM, EDDM, ADWIN

Abordagens passivas
    - incrementam e atualizam os modelos sem a detecção explícita do drift.
        - ARF, Leveraging Bagging
    - abordagens híbridas
        - seleção dinâmica de classificadores. ^7vQTup9R

Frameworks ^EHw67e5p

River (python), Scikit-Multiflow, MOA (CapyMoa). ^7540grRX
